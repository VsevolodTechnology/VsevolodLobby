# Session audit — 2026-05-23

Большая итерация по лобби: миграция формата текста, ребрендинг, новая система серверов
и меню, паркур-фичи, новые админ-команды и серия UX-полировок. Ниже — что менялось и зачем.

## 1. Платформа текста — MiniMessage везде

- [Text.java](src/main/java/ua/vsevolod/lobby/util/Text.java) переписан как
  MiniMessage-движок. `Text.c()` сохранил кэш (для статических строк), `Text.raw()` —
  без кэша (для динамических). Legacy `&`/`§`-коды и `&#RRGGBB` транслируются в
  MiniMessage-теги на лету — старые конфиги не ломаются.
- Все исходники проекта (≈190 замен в 34 файлах) переведены с legacy на MiniMessage.
  В коде больше нет ни одного `<gold>`/`<yellow>`/`&f`. `&f` запрещён —
  «белый» проекта = `<#FFF2E0>`.
- [CommandsReferenceWriter](src/main/java/ua/vsevolod/lobby/feature/admin/config/CommandsReferenceWriter.java)
  переписан: справочник для админов теперь документирует MiniMessage, а не `&`-коды.
- Зависимость `net.kyori:adventure-text-minimessage:4.26.1` добавлена в
  [build.gradle.kts](build.gradle.kts) явно.

## 2. Полная миграция конфигов на ConfigLib

- Удалены устаревший `ConfigManager` / `ConfigSection` и все 10 `*ConfigSection.java`.
- Все конфиги — `@Configuration`-классы ConfigLib c полем `@Comment`-документацией:
  `TabConfig`, `SidebarConfig`, `WelcomeConfig`, `MenusConfig`, `HologramsConfig`,
  `NpcsConfig`, `JoinItemsConfig`, `QrCardConfig`, `CutsceneConfig`,
  `PlayerBehaviorConfig`, `ServersConfig`.
- Новый реестр [ConfigReload](src/main/java/ua/vsevolod/lobby/config/ConfigReload.java)
  заменяет «забытое» `/reload`: каждый конфиг при `init()` регистрирует свой
  лоадер; команда `/reload` вызывает `ConfigReload.reloadAll()` и отдаёт отчёт
  «загружено N, ошибки в …».
- [ReloadCommand](src/main/java/ua/vsevolod/lobby/command/admin/ReloadCommand.java)
  больше не зависит от ConfigManager — работает только через ConfigReload.

## 3. Ребрендинг → Orjus / `ᴏʀᴊᴜꜱ-ꜱᴛᴜᴅɪᴏ`

- `LobbyConfig.Project`: `NAME`, `SOCIAL_LINKS`, `WEBSITE`, `DISCORD`, `SERVER_IP` —
  все перенесены на Orjus. Добавлены `BRAND` (`ᴏʀᴊᴜꜱ` в малых капителях),
  `BRAND_FULL`, `BRAND_GRADIENT`.
- Удалены все `OVERDYN`-литералы в видимых местах: TAB, sidebar, welcome,
  hologram, kick-сообщения.
- Введена палитра «3 цвета сервера»:
  `#AE3AF3` (яркий), `#985DBC` (средний), `#C58AF0` (лаванда). Скрипт
  массовой перекраски заменил весь оранжево/жёлтый набор по всему проекту
  (181 замена в 28 файлах).

## 4. Система серверов (`config/servers.yml`)

Новые файлы:
- [ServersConfig](src/main/java/ua/vsevolod/lobby/config/server/ServersConfig.java) —
  список серверов, шаблон названия и описания предмета, цвета бара онлайна,
  per-server `itemName`/`itemLore` override.
- [ServerEntry](src/main/java/ua/vsevolod/lobby/config/server/ServerEntry.java) —
  одна запись сервера.
- [ServerConnector](src/main/java/ua/vsevolod/lobby/feature/lobby/ui/menu/ServerConnector.java) —
  единая логика подключения: статус-гейт + сообщения OFFLINE/LOADING/ONLINE/SOON.
- [ServerItemRenderer](src/main/java/ua/vsevolod/lobby/feature/lobby/ui/menu/ServerItemRenderer.java) —
  рендер предмета сервера, общий для конфиг-меню и легаси-меню.
- [ServerInfo](src/main/java/ua/vsevolod/lobby/config/server/ServerInfo.java) теперь
  знает `effectiveStatus()` (ручной флаг + живой опрос прокси).

Изменения:
- [MenuItem](src/main/java/ua/vsevolod/lobby/feature/lobby/ui/menu/config/MenuItem.java)
  получил поле `server-id` — привязка предмета меню к серверу.
- [MenuManager](src/main/java/ua/vsevolod/lobby/feature/lobby/ui/menu/MenuManager.java)
  рендерит привязанные server-предметы и **обновляет их вживую раз в секунду**
  (статус/онлайн меняются на лету). Клик идёт через `ServerConnector`.
- В дефолтах оставлен один сервер: `adventur` («Выживание»), без анархии.

## 5. NPC = моб со скейлом

- `NpcDefinition` расширен полями `entityType` (mob-key или `null` =
  обычный манекен) и `scale`. `LobbyNpc` теперь умеет спавниться любым
  `EntityType` и применять `Attribute.SCALE`; для манекена — прежняя ветка.
  Установлен `setInvulnerable(true)`.
- Отдельный `DecorMobsConfig`/`DecorMobService` (был промежуточным решением) —
  удалён. Декоративные мобы теперь это просто записи в `npcs.yml`.
- В дефолтах: NPC `mode-selector` — **Warden scale 1.15**, кликабельный,
  открывает меню (`[menu] mode-selector`). Allay убран — он по ваниле
  забирает предмет из руки игрока.

## 6. Голограммы

- Удалены отдельный `WelcomeHologramConfig`/`LobbyWelcomeHologramService` и
  соответствующий файл конфига. Голограмма приветствия теперь — обычная
  запись `welcome` в общей `holograms.yml`.
- [HologramManager](src/main/java/ua/vsevolod/lobby/feature/lobby/ui/hologram/HologramManager.java)
  поддерживает плейсхолдер `{online}` и обновляет такие голограммы каждые 5 сек.
- Голограмма `mode-selector` (рядом с NPC выбора режима) — добавлена в дефолты.

## 7. Toggle-items в `join-items.yml`

- Удалён отдельный `ToggleItemsConfig`. Музыка / скорборд / видимость игроков
  теперь лежат под `toggleItems` в `join-items.yml`.
- [ToggleItemDefinition](src/main/java/ua/vsevolod/lobby/feature/lobby/player/join/items/ToggleItemDefinition.java) +
  [ToggleItems](src/main/java/ua/vsevolod/lobby/feature/lobby/player/join/items/ToggleItems.java) —
  чистая модель с материалом on/off, названием, меткой `[Вкл]/[Выкл]`,
  lore-шаблоном с `{status}` и сообщениями для чата.
- `LobbyMusicManager`, `SidebarToggle`, `PlayerHider` переведены на конфиг —
  захардкоженные пары on/off-предметов больше не строятся в коде.

## 8. TAB-лист (`tab.yml`)

- Mspt-блок удалён полностью (и у игроков, и у админов).
- Подвал — компактная одна центральная строка со статами: `онлайн N · пинг Nмс · время T`,
  ниже — кликабельный сайт.
- Шапка — бренд (градиент) + слоган.
- `{state}` — слово-состояние сервера (отличное / хорошее / стабильное /
  высокая нагрузка) по mspt; пороги и подписи настраиваются в `tab.yml`.
- `{time}` — настраиваемый часовой пояс `timeZone` (по умолчанию `Europe/Moscow`)
  + подпись `timeZoneLabel` (по умолчанию `МСК`).

## 9. Per-player время по IP

- [PlayerTimeZoneService](src/main/java/ua/vsevolod/lobby/feature/lobby/player/time/PlayerTimeZoneService.java) —
  при входе асинхронно резолвит IP игрока через `ip-api.com` → `ZoneId`,
  кэширует. Список игроков, кто включил режим «по IP» персистится в
  `storage/time-by-ip.txt` — без правки `PlayerPreferences`/Mongo/файлового
  стора.
- В [LobbySettingsMenu](src/main/java/ua/vsevolod/lobby/feature/lobby/ui/menu/LobbySettingsMenu.java)
  добавлен предмет «Время по IP» с честным предупреждением: пояс определяется
  по IP, при VPN может быть неточным.
- В TAB-подвале `{time}` подставляется per-player: либо пояс игрока (UTC-offset
  в скобках), либо `timeZone` сервера.

## 10. Паркур — фичи

- Дефолт при старте: **Незер + Соревновательный** (раньше Overworld + Normal).
- Кратные титулы прогресса каждые 10 очков (быстрый, нелезущий в глаза
  титул/субтитул в духе «Красава! / так держать темп»).
- Соревновательный — сообщения «Ты обогнал X — N очк.» при обходе игрока
  лидерборда.
- **Возврат на пред-паркурную позицию.** При старте сохраняется поза игрока,
  при завершении/падении/выходе он возвращается на это место, а не на спавн
  (см. [LobbyParkourService](src/main/java/ua/vsevolod/lobby/feature/lobby/interaction/parkour/LobbyParkourService.java)).
- **Сохранение выбора паркура между перезапусками.** Сложность / измерение /
  тренировка / звук пишутся в `storage/parkour-prefs.properties` —
  `ParkourSettingsMenu` лениво подгружает их при первом обращении.

## 11. Команды

- Новая [`KickCommand`](src/main/java/ua/vsevolod/lobby/command/admin/KickCommand.java):
  `/kick <ник> [причина]` с брендированным экраном отключения. Только для
  операторов; защита от self-kick и от пустого ника.
- `[connect]` и `transfer-server`-action в `LobbyModule` перенесены на
  `ServerConnector` — единая логика подключения.

## 12. UX-полировки приветствия и социалок

- Приветствие в чате ужалось до трёх строк (раньше уходило за край экрана).
  Соцсети — кликабельные слова с hover-подсказкой, без сырых URL.
- QR-карта (`qr-card.yml`) — заголовок-градиент «Наши социальные сети»,
  строки ссылок одинаковым стилем с подсказками: Discord / Telegram / Сайт.
- `SocialLink` в QR-конфиге расширен полями `color` и `hint`.
- Баг с «жирность течёт на всю строку» (незакрытый `<bold>` в шаблоне)
  устранён.

## 13. Графика и заголовки

- Per-char-«градиенты» (`<#hex><bold>X<#hex><bold>Y…`) — артефакт массовой
  перекраски, при котором первая буква оказывалась ярче, а остальные —
  одинакового тёмного оттенка — схлопнуты в нормальный
  `<gradient:from:to><bold>WORD</bold></gradient>` (18 строк в 9 файлах).
- Голограмма паркура переоформлена: плавный градиент-заголовок, цветные
  акценты, без тяжёлого жирного.
- Голограмма выбора режима — короткая, 2 строки.
- Mode-selector меню — фиолетовая рамка, header «Игровые режимы» с описанием
  в стиле «`«Что это» — «Как играть»»`, заглушки «Новый режим (Скоро)»,
  заголовок чарта — фиолетовый градиент.
- Sidebar: серый цвет заменён на лавандово-серый `#9C93B0`, все
  `<gray>`/`<gold>`/`<yellow>` — на покет покраски.
- Welcome-голограмма (общая) — обновлённый текст под бренд Orjus,
  плейсхолдер `{online}` рендерится с живым счётчиком.
- Welcome-анимация (титул на входе) фиксит литеральный `<bold>` и теперь
  правильно извлекает декорации через MiniMessage-парсинг (а не legacy-strip).

## 14. Системные сообщения / Messages

- `Messages.WARNING_COLOR` — янтарный `#E8A030` → фиолетовый `#D9A6F0`.
  `Messages.ACCENT_COLOR` — лаванда. Префикс `[Система]` — фиолетовый,
  не оранжевый.
- `LobbyPlayerLoginListener` — kick-сообщения пропущены через `Text.raw`,
  чтобы теги парсились.

## 15. Удалено / переименовано

- Удалены: `ConfigManager`, `ConfigSection`, все `*ConfigSection.java`,
  `WelcomeHologramConfig`, `LobbyWelcomeHologramService`, `WelcomeHologram`,
  `DecorMobsConfig`, `DecorMobService`, `DecorMob`, `ToggleItemsConfig`,
  `ToggleItemDefinition` (старый в `config/`), `JumpPattern`.
- `JOIN_ITEM_ID` блокируется в `LobbyBlockProtectionListener` — даже админ
  не может поставить join-item в мир.

## 16. Дефолты конфигов, требующие удаления `*.yml` на сервере

Поскольку ConfigLib не перезаписывает существующие файлы при изменении
дефолтов, следующие файлы регенерируются при удалении и применяют новые
дефолты (это делается автоматически на каждом деплое):

`tab.yml`, `sidebar.yml`, `welcome.yml`, `menus.yml`, `holograms.yml`,
`npcs.yml`, `servers.yml`, `join-items.yml`, `qr-card.yml`.

Не трогаются: `proxy.yml` (порт/адрес/секрет), `storage.yml` (Mongo URI),
`cutscene.yml` (вейпоинты от `/cutscene add`), `logging.yml`, `restart.yml`,
`chat.yml`, `player-behavior.yml`.

---

Всё собирается shadowJar-ом, без warning-ов уровня error. Деплой — SFTP в
`/server.jar`, рестарт лобби — ручной через панель (RCON 50011 недоступен
из окружения сборки).
